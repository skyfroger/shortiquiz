# Shortiquiz

**Shortiquiz** - расширение для [Quarto](https://quarto.org/), которое позволяет добавлять на страницы сайта интерактивные тестовые задания: вопросы с выбором ответа, текстовые поля, задачи Парсона, флэшкарты и многое другое.

## Установка

Добавьте расширение в свой проект Quarto:

```bash
quarto add skyfroger/shortiquiz
```

После установки шорткоды станут доступны сразу, а фильтры - после подключения `shortiquiz` к проекту или отдельной странице.

## Локализация

Для локализации элементов интерфейса расширения, укажите в YAML атрибут `lang`. По умолчанию используется английский язык. Также доступен русский язык:

```yaml
lang: ru
```

## Шорткоды

Шорткоды позволяют встраивать интерактивные элементы прямо в текст абзаца.

### `qselect` - выпадающий список

Добавляет выпадающий список с вариантами ответов. Правильным считается **первый** указанный вариант; перед показом варианты перемешиваются.

**Синтаксис:**

```markdown
{{< qselect "вариант1|вариант2|вариант3" show="текст" mono=true >}}
```

**Параметры:**

| Параметр | Описание |
|----------|----------|
| *(первый аргумент)* | Варианты ответов, разделённые `\|`. Первый - правильный. |
| `show` | Текст, который отобразится после правильного ответа. Необязательный. |
| `mono` | Если `true`, ответ выводится моноширинным шрифтом. |

> **Цвет ответа:** если правильный ответ получен с первой попытки, текст выделяется **зелёным**; иначе - **синим**.

**Пример:**

```markdown
В Python функция для преобразования в целое число - это {{< qselect "int()|float()|str()" show="int()" mono=true >}}.
```

### `qinput` - текстовое поле

Добавляет текстовое поле для ввода ответа. Проверка происходит по нажатию `Enter` или при потере фокуса.

**Синтаксис:**

```markdown
{{< qinput ответ1|ответ2|... hints="подсказка" show="текст" tol=число size=3 mono=true >}}
```

**Параметры:**

| Параметр | Описание |
|----------|----------|
| *(первый аргумент)* | Правильные варианты ответа, разделённые `\|`. Первый используется для отображения, если не задан `show`. |
| `tol` | Допуск для числовых ответов. Правильным считается значение в диапазоне `n - tol <= ответ <= n + tol`. |
| `show` | Текст, показываемый вместо поля после правильного ответа. |
| `hints` | Подсказка, которая появляется после трёх неправильных попыток. |
| `mono` | Моноширинный шрифт для отображения правильного ответа. |
| `size` | Ширина поля ввода (атрибут `size` тега `<input>`). По умолчанию `2`. |

**Пример 1 - текстовый ответ:**

```markdown
Рассмотрим ШИМ-сигнал с коэффициентом заполнения 50%. Если опорное напряжение равно 5 В, то среднее напряжение на выходе будет равно {{< qinput 2.5|2,5 hints="50% от значения напряжения" show="два с половиной" size=3 >}} Вольта.
```

**Пример 2 - числовой ответ с допуском:**

```markdown
Введите число от 8 до 12: {{< qinput 10 tol=2 >}}.
```

## Фильтры (блочные вопросы)

Для использования блочных вопросов необходимо подключить фильтр `shortiquiz` (ко всему проекту или отдельной странице).

### Вопрос с множественным выбором (`qmulti`)

Вопрос оформляется внутри блока с классом `.qmulti`. Весь текст до списка - формулировка вопроса, пункты списка - варианты ответов. **Первый пункт считается правильным.** Варианты перемешиваются при показе.

**Минимальная разметка:**

```markdown
:::{.qmulti}

Как можно определить тип переменной?

- Использовать функцию `type`.
- Вывести значение переменной на экран и определить её тип по этому результату.
- Использовать её в выражении, значение которого известно.
- Посмотреть на описание переменной.

:::
```

**Подсказки и отзывы:**

- Блок-цитата `>` **перед** списком - подсказка к вопросу (доступна по кнопке с вопросом после неправильного ответа).
- Блок-цитата `>` **внутри пункта списка** - отзыв на конкретный вариант ответа.

**Пример с подсказками и отзывами:**

````markdown
:::{.qmulti}

Какое значение выведется на экран:

```python
print(int(53.785))
```

> Как проще всего из целого числа получить вещественное?

- 53

  > Число 53.785 преобразуется в целое функцией `int`. Дробная часть исходного числа отбрасывается.

- Возникнет ошибка времени выполнения.

  > В этом выражении нет синтаксических ошибок.

- 54

  > Функция `int` не округляет вещественное число.

- 53.785

  > Функция `int` преобразует число в целое.

:::
````

### Открытый вопрос (`qinput`)

Блок с классом `.qinput` используется для вопроса с открытым ответом. Первый пункт списка - правильный ответ.

**Пример:**

````markdown
:::{.qinput}

Каким будет результат выполнения кода:

```python
print(42)
```

> Обратите внимание на то, что написано в скобках.

- 42

- 24

  > Цифры перепутаны местами.

:::
````

### Вопрос с несколькими правильными ответами (`qcheck`)

Блок с классом `.qcheck` содержит чек-лист (`- [ ]` / `- [x]`). Правильные варианты помечаются крестиком `x`.

**Минимальная разметка:**

```markdown
:::{.qcheck}

Выберите варианты с корректным именем переменной в Python.

- [x] `sum`
- [ ] `1st_name`
- [ ] `class`
- [x] `_last_name`
- [x] `Student`

:::
```

**Пример с отзывами:**

```markdown
:::{.qcheck}

Что из перечисленного является логическим выражением?

- [x] `True`

  > `True` и `False` - логические литералы. Их можно отнести к логическим выражениям.

- [x] `3 == 4`

  > Результат сравнения с помощью `==` принимает значение `True` или `False`.

- [ ] `3 + 4`

  > Значение этого выражения - число.

- [x] `3 + 4 == 7`

  > Выражение 3+4 равно 7. `7 == 7` - логическое выражение.

- [ ] `"False"`

  > Значение в двойных кавычках, значит это строка.

:::
```

### Задачи Парсона (`qparson`)

Задание, в котором нужно расположить строки кода в правильном порядке.

**Атрибуты:**

| Атрибут | Описание |
|---------|----------|
| `spaces` | Количество пробелов в отступе. По умолчанию `4`. |
| `sep` | Произвольный сепаратор для объединения соседних строк в один блок инструкций. |

**Пример базовой разметки:**

````markdown
:::{.qparson spaces=4}

Элементы, не являющиеся блоками кода, считаются условием задачи.

Это могут быть изображения, диаграммы, таблицы.

```python
n = int(input('Введите число'))
while n > 0:
    print(n)
    n = n - 1
```

:::
````

**Объединение строк в блоки:**

Если порядок следования нескольких инструкций не важен, их можно объединить с помощью сепаратора `sep`:

````markdown
:::{.qparson sep=##}

```python
n = int(input("n>"))
sum = 0##
while n > 0:##
    if n % 2 == 0:##
        print(n)
        n = n - 1##
```

:::
````

**Добавление дистракторов:**

Неправильные варианты (дистракторы) помещаются во **второй** блок кода внутри `.qparson`:

````markdown
:::{.qparson spaces=4 sep=##}

```python
n = int(input())
sum = 0##
while n > 0:##
    if n % 2 == 0:##
        print(n)
        n = n - 1##
```

```py
n = int(input())
sum = 1##
```

:::
````

### Поиск области на изображении (`qspot`)

Пользователю предлагается перетащит пронумерованные маркеры на корректные области изображения.

**Атрибут `pos`** задаёт координаты области в формате `"x y width height"`.

> **Примечание:** для корректной работы демо замените `example.assets/img.png` на реальный путь к изображению в вашем проекте.

```markdown
:::{.qgroup}

:::{.qspot}

![](images/img.png)

Найдите следующие элементы на изображении:

[фоторезистор]{pos="57 24 12 12"}

[красный светодиод]{pos="43 61 13 31"}

[микроконтроллер]{pos="1 41 58 42"}

> Подсказка к вопросу.

:::
```

## Дополнительные возможности

### Группировка вопросов (`qgroup`)

Несколько идущих подряд вопросов `qmulti`, `qcheck`, `qinput`, `qparson` и `qspot` можно объединить в группу:

```markdown
:::{.qgroup}

:::{.qmulti}
...
:::

:::{.qcheck}
...
:::

:::
```

### Разблокировка этапов (`qgate` + `qnext`)

Пошаговые инструкции, где каждый следующий этап открывается только после нажатия кнопки.

**Разметка:**

```markdown
::::{.qgate name=s1}

... содержимое первого этапа ...

::::

::::{.qgate name=s2}

Следующий этап открывается по кнопке:

{{< qnext gate=s2 >}}

::::

::::{.qgate name=s2}

Это последний этап инструкции.

::::
```

> **Важно:** первый этап (`s1`) не требует кнопки и отображается сразу. Кнопка `qnext` с `gate=s2` размещается внутри предыдущего этапа (`s1`) и открывает следующий.

### Подсказки к решению (`qsolution`)

Блок `.qsolution` содержит последовательные подсказки к решению задачи. Каждая подсказка - элемент нумерованного или ненумерованного списка.

- Подсказки открываются **последовательно** (по одной).
- Если в блоке есть блок с кодом, он считается **решением задачи**.
- Решение доступно только после открытия всех подсказок.
- Строки кода решения изначально скрыты и открываются по одной; после десятой строки можно открыть код целиком.

**Пример:**

````markdown
:::{.qsolution}

1. Используйте каскадную форму записи инструкции ветвления.
2. Учтите, что к зимним относятся месяцы с номерами 1, 2 и 12.
3. Достаточно прописать логические выражения для трёх пор года. Если ни одно из трёх условий не выполнится, значит выполнится ветвь `else` для оставшейся последней поры года. Так получится сократить код.

```python
month = int(input("Введите номер месяца (от 1 до 12): "))

if month == 1 or month == 2 or month == 12:
    print("Зима")
elif month >=3 and month <=5:
    print("Весна")
elif month >= 6 and month <= 8:
    print("Лето")
else:
    print("Осень")
```

:::
````

### Флэшкарты (`qflashcards`)

Набор карточек с вопросами и ответами. Учащийся читает вопрос, вспоминает ответ, затем проверяет себя. Если ответ не удалось вспомнить, карточка добавляется в конец стопки.

**Разметка:**

```markdown
:::{.qflashcards}

- Функция для вывода значений на экран.

  > `print()`

- Функция для ввода значений с клавиатуры.

  > `input()`

- Значения какого типа возвращает функция `input()`?

  > Строкового (`str`)

- Функция для преобразования значений в целые числа.

  > `int()`

:::
```

> Элемент цитаты `>` внутри пункта списка содержит **ответ**. Остальные элементы - формулировка вопроса.

### Карточка с вопросом (`qflip`)

Интерактивная карточка с двумя сторонами: вопросом и ответом. Клик по карточке переворачивает её.

**Разметка:**

````markdown
:::{.qflip}

С помощью какой функции можно прочитать значение с аналогового пина Arduino?

---

```c++
analogRead(пин);
```

:::
````

> Вопрос и ответ разделяются горизонтальной линией `---`.

## Советы по разметке

1. **Правильный ответ всегда первый.** В `qmulti`, `qinput` (блочный) и `qselect` первый указанный вариант считается правильным. В `qcheck` правильность задаётся через `[x]`.
2. **Перемешивание.** В `qmulti` и `qselect` варианты ответов автоматически перемешиваются перед показом.
3. **Подсказки и отзывы.** Используйте блоки-цитаты `>` для добавления пояснений. Расположение цитаты (перед списком или внутри пункта) определяет её назначение.
4. **Код внутри вопросов.** Quarto поддерживает синтаксис подсветки кода внутри блоков вопросов - используйте fenced code blocks (```` ``` ````).
5. **Вложенность блоков.** Для `qgroup` и `qgate` используйте 4 двоеточия (`::::`) для внешнего блока, чтобы избежать конфликтов с внутренними блоками из 3 двоеточий (`:::`).
